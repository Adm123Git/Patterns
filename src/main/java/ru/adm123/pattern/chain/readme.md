**Паттерн "CHAIN"**
***
Используется для обеспечения гибкости преобразования или проверки данных.<br>
По смыслу немного похож на <b>BUIDER</b>, но работает с готовыми объектами.
***
chain предполагает создание нескольких классов-обработчиков данных вместо одного большого метода.

Каждый из этих классов проводит свою проверку/преобразование и опционально вызывает следующий. Получается цепочка вызовов,
что позволяет проводить преобразование/проверку в любом порядке и в любом объеме в зависимости от того, как построена цепочка.
***
