**Паттерн "DECORATOR"**
***
Используется для создания объекта на основе некоего базового объекта, при этом добавляя/изменяя функционал базового объекта
***
**_decorator_** предполагает создание классов-оберток, наследующих базовую сущность (или имплементящих базовый интерфейс),
которые могут оборачивать ("декорировать") базовую сущность и таким образом как-то менять функционал объекта
(добавлением и/или оверрайдом методов).
